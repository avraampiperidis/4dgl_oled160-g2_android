Android notifications to mOled-160-G2 via bluetooth.

A very simple step by step guide/tutorial to create a wireless(bluetooth) 1.7 inch monitor, an android app to connect to it,and automatically send every notifications from android to monitor.

required components

1)uOLED-160G2 Display Module display module
2)uUSB-PA5-II Programming Adaptor adapter
3)Bluetooth module
4)Programming Cable
5)USB to 4-pin cable
6)4-pin to 4-pin cable
7)Any USB power bank
8)Workshop4 IDE link
9)Android Studio

Now the whole bluetooth initialization and connection to the remote bluetooth is happening inside the onCreate() method witch is run when user gives permission.
basically we this global variables

private static BluetoothAdapter mBluetoothAdapter;
private static BluetoothSocket mmSocket;

The important stuff at onCreate. first we get a reference to the bluetooth adapter

 mBluetoothAdapter = BluetoothAdapter.getDefaultAdapter();
        //bluetooth hardware is absent
        if(mBluetoothAdapter == null) {
            makeText("Device does not support Bluetooth! Exiting!");
            return;
        }


then we get the devices and we search for the remote bluetooth we want device.getName().startsWith("HC") HC is the name for my bluetooth

Set<BluetoothDevice> pairedDevices = mBluetoothAdapter.getBondedDevices();
            for (BluetoothDevice device : pairedDevices) {
                //HC is the remote bluetooth module
                if (device.getName().startsWith("HC")) {
                    mmDevice = device;
                    break;
                }
            }


As we found the bluetooth , we then connect to it.

//Standard //SerialPortService ID //we need this
UUID uuid = UUID.fromString("00001101-0000-1000-8000-00805f9b34fb");
mmSocket = mmDevice.createRfcommSocketToServiceRecord(uuid);
mmSocket.connect();
mmOutputStream = mmSocket.getOutputStream();


Thats it! The android bluetooth is now connected to the remote bluetooth. And we can start sending bytes. We can send bytes from the mmOutputStream object.
We start by sending first our network carrier name.

TelephonyManager manager = (TelephonyManager) getSystemService(Context.TELEPHONY_SERVICE);

And the results in two pictures.

for the time i have a thread running forever and sending time every one minute. the time wont be sent if the monitor has an notification and the user hasn't read it.

//every 1 minute send time
//better to change to global variable thread
        new Thread() {
            public void run() {

                while (true) {

                    try {
                        //60000 millis = 1 min
                        Thread.sleep(60000);
                        //if false means user read/cleaned all notifications
                        if (notifyRead == false) {

                            Calendar c = Calendar.getInstance();
                            int minutes = c.get(Calendar.MINUTE);
                            int hour = c.get(Calendar.HOUR);
                            String time = hour + ":" + minutes;

                            try {
                                mmOutputStream.write('~');
                                mmOutputStream.write(time.getBytes());
                            } catch (IOException e) {
                                e.printStackTrace();
                            }
                        }

                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }

                }
            }
}.start();

Next we override this method onNotificationRemoved.
This method gets called when the user cleans the notifications.

 @Override
    public void onNotificationRemoved(StatusBarNotification sbn) {
        if(inputKeyboard == true) {
            inputKeyboard = false;
        } else {
            try {
                notifyRead = false;
                mmOutputStream.write('`');
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
}

basically the notifyRead = false means the user have read the notification

From the above ween to remember/know the following:
*Pin 1:Input voltage supply,this is where the display taking power

**Pin 2:Output voltage, When we connect the bluetooth module to the display
it need power to work, this is where it gets it.
Pin 5:RX, this receives the data/bytes from the bluetooth module.
Pin 7:Ground for usb power bank
Pin 8:Ground for bluetooth module
