Android notifications to mOled-160-G2 via bluetooth.

A very simple step by step guide/tutorial to create a wireless(bluetooth) 1.7 inch monitor, an android app to connect to it,and automatically send every notifications from android to monitor.

required components

1)uOLED-160G2 Display Module display module
2)uUSB-PA5-II Programming Adaptor adapter
3)Bluetooth module
4)Programming Cable
5)USB to 4-pin cable
6)4-pin to 4-pin cable
7)Any USB power bank
8)Workshop4 IDE link
9)Android Studio

Starting from Android Studio and the app

I will post small snippets of the actual code. Clone this git to get the whole project.

first some important stuff about AndroidManifest.xml
 <uses-permission android:name="android.permission.READ_PHONE_STATE" />
    <uses-permission
        android:name="android.permission.BLUETOOTH" />

android:name="android.permission.BLUETOOTH_ADMIN" />

android.permission.READ_PHONE_STATE to get the carrier network android.permission.BLUETOOTH_ADMIN to get access to turn off n on the bluetooth 

 <service android:name="com.example.abraham.iNotify.NLService"
            android:label=".NLService"
            android:permission="android.permission.BIND_NOTIFICATION_LISTENER_SERVICE">
            <intent-filter>
                <action android:name="android.service.notification.NotificationListenerService" />
            </intent-filter>
</service>

We declare our notification service class. The service class is the NLService witch we will create later. This service will run in the background,and will receive every notification.

Basically its only three classes,MainActivity,NLService,Utilities NLService its the notification service. MainActivity its the app's main activity with some buttons Utilities contains some string n utils methods

MainActivity
@Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        Intent intent = new Intent("android.settings.ACTION_NOTIFICATION_LISTENER_SETTINGS");
        startActivity(intent);
}

Note this lines: Intent intent = new Intent("android.settings.ACTION_NOTIFICATION_LISTENER_SETTINGS"); startActivity(intent);
this is needed to start an activity and the user can give permission to NLSerivce to run.
we need to mention the below method createNotification() witch can create notifications for test.

public void createNotification(View v) {
        NotificationManager nManager = (NotificationManager) getSystemService(NOTIFICATION_SERVICE);
        NotificationCompat.Builder ncomp = new NotificationCompat.Builder(this);
        ncomp.setContentTitle("My Notification Title");
        ncomp.setContentText("Notification example");
        ncomp.setTicker("Notification LExample");
        ncomp.setSmallIcon(R.drawable.ic_launcher);
        ncomp.setAutoCancel(true);
        nManager.notify((int)System.currentTimeMillis(),ncomp.build());
}


and i have a sendMessage method witch takes the text from edittext and send's it to outputstream "bluetooth at our case"

public void sendMessage(View view) throws IOException {
        EditText editText = (EditText) findViewById(R.id.edit_message);
        String textToSend = editText.getText().toString();
        NLService.sendMessage(textToSend);
}
